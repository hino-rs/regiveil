# Windows Registry 用語集

## 基本構造に関する用語

レジストリは、Windowsのエクスプローラーのようなツリー上の階層構造を持つ。

- Key
  - エクスプローラーにおける「フォルダ」相当。設定項目をグループ化して整理するために使われる。
- Subkey
  - あるキーの中に入れ子になっている別のキーのこと。
- Value
  - エクスプローラーにおける「ファイル」相当。実際の設定データが格納されている。
  - 値は「名前」「種類」「実際の設定値」の3つの要素で構成されている。
- Hive
  - レジストリのツリー構造の一部を構成する、物理的なファイル(HDD/SSD上に保存されている実体)のまとまりのこと。

## ルートキー

レジストリの最上位にある5つの大元となるキー。全て「`HKEY_`」から始まる。

- `HKEY_CLASSES_ROOT`(HKCR)
  - ファイルの拡張子と、それを開くアプリケーションの関連付けなどの情報が保存されている。
- `HKEY_CURRENT_USER`(HKCU)
  - 現在Windowsにサインインしているユーザー個人の設定(e.g.,デスクトップ背景)が保存されている。
- `HKEY_LOCAL_MACHINE`(HKLM)
  - ユーザーに関係なく、そのパソコン全体に適用される設定が保存されている。
- `HKEY_USERS`(HKU)
  - そのパソコンに登録されている全てのユーザーのプロファイル情報が保存されている。HKCUは、ここの一部を読み込んで表示している。
- `HKEY_CURRENT_CONFIG`(HKCC)
  - 起動時に読み込まれた現在のハードウェアプロファイルが保存されている。

## 値の種類

「値のデータ」として入力できる形式。以下は、よく使われる代表的な形式。

- REG_SZ: 文字列
- REG_DWORD: 32bit bin
- REG_QWORD: 64bit bin
- REG_BINARY: hex
- REG_MULTI_SZ: 複数行文字列
- REG_EXPAND_SZ: 展開可能な文字列
  - `%USERPROFILE%`や`%SystemRoot%`などの環境変数を含んだ文字列で、OSが読み込む際に実際のフォルダパスに展開される。

## 操作・ツール

- レジストリエディター
  - Windowsに標準搭載されている、レジストリを閲覧・編集するためのツール。
- エクスポート/インポート
  - レジストリの特定のキーや全体を、`.reg`という拡張子のバックアップファイルとして保存(エクスポート)したり、そのファイルを読み込んで復元(インポート)したりする機能。
